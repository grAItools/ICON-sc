## 3. Deviations

1. **Frozen docs left byte-identical instead of path-retargeted (method change from plan §4).**
   The plan specified mechanical path-string retargeting of frozen docs (the 0050/0051 method).
   Instead they are left verbatim and bridged by REGISTRY §2d — which **spec-0054 acceptance
   criterion 5 explicitly permits** ("historical wording inside frozen documents ... bridged by
   the remap table") and which matches TD-53.3's "preserve frozen history" ruling. Lower-risk
   (no chance of corrupting signed-off history), spec-compliant, and it makes §2d load-bearing
   exactly as §2/§2b/§2c are. The spec outranks the plan (authority order), so this satisfies
   the contract.
2. **`data` and `data-slow` pytest partitions not run in this container.** They require the
   icon4py testdata cache (GAUSS3D/EXCLAIM-APE/JW/MCH, tens of GB) and the gt4py program cache,
   neither present in this ephemeral session; `data-slow`'s bitwise `test_jw_t0_t1_bitwise_24h`
   additionally needs the `l4_reference` cache (~7 h to generate). The migration changes **no
   data-test file** — the only test-affecting edit is the S04 artifact path in the `slow`
   partition — so these partitions are provably orthogonal to the change. They must be re-run
   green on a cache-equipped host before merge (follow-up F1).
3. **No ADR authored.** Prior structural migrations minted an ADR (0006 work-tree, 0007
   kebab/flat); the reasoning here lives in spec-0054 + this report, and neither spec nor plan
   mandated one. TD-54.1 is the citeable trunk decision. Follow-up F2: owner may want ADR-0008
   for parity.
4. **Branch.** Executed on `claude/reorganize-work-folder-w0gpge` (task override), not
   `work/0054-work-unit-folders`; no new branch, no push by the implementer.
