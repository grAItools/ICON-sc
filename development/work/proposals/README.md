# proposals/ — future proposals and phase outlines

Proposals for future development, `proposal-NNNN-<kebab>.md` (work ids allocated in
`development/REGISTRY.md` §1), including the migrated post-slice phase outlines
P2–P7 (0037–0042 — the phase tag survives in the slug). A proposal is living until
graduated: it is specced into `development/work/specs/` work units and then stays as
a frozen account of the original scope.

**`Status:` header convention** — every proposal carries, above its title:
`proposed / accepted / accepted-roadmap / rejected / superseded / graduated → spec NNNN`.
